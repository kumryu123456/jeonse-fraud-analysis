# 전세사기 위험도 분석 시스템

> 서울시 전세 거래 데이터 150,000건과 전세사기 피해 데이터를 기반으로 구별 위험도를 분석하는 데이터 분석 프로젝트

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **기간** | 2024.09 - 2024.12 |
| **팀 구성** | 5인 (데이터 분석 담당, 기여도 40%) |
| **과목** | AI와 데이터기초 |
| **소속** | 숭실대학교 전자정보공학부 IT융합 |

## 주요 성과

- 공공데이터포털 API 활용, 전세 거래 데이터 **150,000건** 수집 및 전처리
- IQR 방식 이상치 제거 + 정규식 기반 시군구 데이터 정제
- 피어슨 상관분석 및 OLS 회귀분석으로 **3억~5억 구간 사기 발생률이 평균(1.8%) 대비 3.2배 높은 5.8%** 발견 — 정책 제언 리포트 20페이지 작성
- 구별 위험도 점수 산출 시스템 구현 (보증금 차이 30% + 피해건수 35% + 피해율 35%)

## 기술 스택

- **Language**: Python
- **Data Analysis**: Pandas, NumPy
- **Visualization**: Matplotlib, Seaborn
- **Statistics**: SciPy (Pearson correlation), Statsmodels (OLS regression)

## 분석 파이프라인

```
데이터 수집 → 전처리(정규식, IQR) → 탐색적 분석 → 상관관계 분석 → 회귀분석 → 위험도 스코어링
```

### 1. 데이터 전처리
- CSV 인코딩 처리 (CP949)
- 보증금 컬럼 숫자 변환 및 콤마 제거
- 정규식으로 시군구 정보 추출
- IQR 방식 이상치 제거

### 2. 시각화 분석
- 구별 전세사기 피해건수 바 차트
- 구별 평균 보증금 분포
- 보증금-피해건수 산점도 + 회귀선

### 3. 통계 분석
- 피어슨 상관계수 및 p-value 검정
- OLS 회귀분석
- 보증금 구간별(Quartile) 피해 현황 분석

### 4. 위험도 스코어링
사용자가 구와 보증금을 입력하면 종합 위험도 점수(100점 만점)를 산출합니다.

```
위험도 = 보증금차이(30%) + 피해건수(35%) + 피해율(35%)
```

## 실행 방법

pip install -r requirements.txt
